Scraping React apps without a headless browser used to feel impossible. I needed marketing data that lived inside a handful of modern SPAs, but running a headless browser for every request was painfully slow and expensive. There had to be a better way.

The trick: most SPAs ship all of their data to the browser inside JSON blobs so the client can hydrate instantly. If you can capture those blobs, you can read the same structured data without ever executing the app.

## Why are SPAs harder to scrape?

Traditional sites render HTML on the server, so `GET` requests return DOM-ready data. Single-page apps render on the client, so the initial HTML is mostly a skeleton that runs scripts and fetches data after load. Scraping that with simple requests leaves you with empty divs.

The challenge is that React and Next.js apps still need to ship their data somewhere. They usually embed it as JSON in `<script>` tags or return it from predictable JSON APIs. If we can find that payload, we can parse it directly.

## The Solution

Instead of launching a headless browser, I issue a plain HTTP request and parse the response HTML for embedded data stores. Here's the small helper I use to fetch the page:

```python
import requests

HEADERS = {
    "User-Agent": "Mozilla/5.0 (Macintosh; Intel Mac OS X 14_4_1) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/124.0.0.0 Safari/537.36",
    "Accept": "text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,image/apng,*/*;q=0.8",
}


def fetch(url: str) -> str:
    response = requests.get(url, headers=HEADERS, timeout=30)
    response.raise_for_status()
    return response.text
```

### Extracting React/Next.js Data from HTML

Most frameworks stash their boot data in `<script id="__NEXT_DATA__">` or similar nodes. You can parse those nodes with BeautifulSoup (or another HTML parser) and load the JSON directly:

```python
from __future__ import annotations

import json
from bs4 import BeautifulSoup
from typing import Any


def extract_data_models(html: str) -> list[dict[str, Any]]:
    soup = BeautifulSoup(html, "html.parser")
    data: list[dict[str, Any]] = []

    for script in soup.select('script[id^="__NEXT_DATA__"], script[type="application/json"]'):
        if not script.string:
            continue
        data.append(json.loads(script.string))

    return data
```

This skips the JS execution entirely. You end up with structured dictionaries matching whatever the React app expected.

### Handling Authentication and Cookies

If the site uses authenticated routes, mimic the headers and cookies your browser already sends. I export the relevant session cookie from Chrome, store it as an environment variable, and rehydrate it inside a shared session:

```python
import os
import requests

session = requests.Session()
session.headers.update(HEADERS)
session.cookies.set("sessionid", os.environ["SESSION_ID"])

response = session.get("https://example.com/dashboard", timeout=30)
response.raise_for_status()
```

Duplicating the headers prevents basic bot detection, and setting cookies keeps you within the authenticated experience without recreating a login flow.

## Conclusion

If you only need structured data, scraping SPAs doesn't require puppeteer scripts or play-wrights. Take the JSON the app already ships, parse it, and move on with your life.
