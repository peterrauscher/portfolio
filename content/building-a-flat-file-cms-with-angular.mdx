A flat-file CMS allows you to manage content for your website without a database backend. Instead, store your pages as regular text files (or in this case, Markdown files), build, and publish. If you’re reading this right now, you know what the result of this looks like. I spent the last two days building my own flat-file CMS for my site, and now I’d love to walk you through how I did it.

Though I’ll skip most of the stylistic choices (because they’re mostly irrelevant) this framework can be used to build a custom blog on your own Angular website, without the need to individually create new components for each post, manage a database, or write your posts in HTML format. Let’s get started!

## Converting posts from Markdown to HTML

Writing in Markdown is super convenient, and supported by just about any text editor. To convert these `.md` files to browser-ready HTML, I wrote a simple Node.js script using two great npm packages called [gray-matter](https://www.npmjs.com/package/gray-matter) and [showdown](https://www.npmjs.com/package/showdown).

The `gray-matter` package lets us define Jekyll-style front matter for our posts, so we can add metadata for each that we’ll use later in the build script — like so:

```yaml
---
title: Some blog post title
date: 2020-01-24
published: true
thumbnail: some-pic.jpg
permalink: some-post
---
```

The `showdown` package converts the Markdown content to HTML. For example, this Markdown:

```
Some blog post introduction...
## A sub-heading
Some section content...
```

Will look like this:

```html
<p>Some blog post introduction...</p>
<h2>A sub-heading</h2>
<p>Some section content...</p>
```

First, the script finds all of the relevant files. It checks the `posts` folder, as defined by the `MARKDOWN_DIRECTORY` variable:

```js
const MARKDOWN_DIRECTORY = path.join(__dirname, "posts");
```

Next, it reads each Markdown file into a global array of objects called `files`:

```js
let files = [];
files = fs
  .readdirSync(MARKDOWN_DIRECTORY)
  .map((f) => path.join(MARKDOWN_DIRECTORY, f))
  .filter((f) => fs.lstatSync(f).isFile())
  .map((file) => {
    // Read the whole file to a string
    let fileContent = fs.readFileSync(file).toString();
    // Parse it with gray-matter
    let post = matter(fileContent);
    post.filename = file;
    // Convert the content to HTML with showdown
    post.content = converter.makeHtml(post.content);
    return post;
  });
```

The line `let post = matter(fileContent)` parses the file content with front matter into an object that looks like:

```json
{
  "data": {
    "title": "Some blog post title",
    "date": "2020-01-24T00:00:00.000Z",
    "published": true,
    "thumbnail": "some-pic.jpg",
    "permalink": "some-post"
  },
  "content": "Some blog post introduction...\n## A sub-heading\nSome section content..."
}
```

…and `converter.makeHtml(post.content)` converts the `content` field to HTML, as you’d expect:

```json
{
  "data": {
    "title": "Some blog post title",
    "date": "2020-01-24T00:00:00.000Z",
    "published": true,
    "thumbnail": "some-pic.jpg",
    "permalink": "some-post"
  },
  "content": "<p>Some blog post introduction...</p>\n<h2>A sub-heading</h2>\n<p>Some section content...</p>"
}
```

Since I wanted thumbnails for each post (the file it uses is defined by the `thumbnail` field in the front matter), I added some code that copies the files from the `posts` folder to the `assets` folder in Angular. Some of the image files I downloaded from [Unsplash](https://unsplash.com) were huge and took a few seconds to load, so I delegated this task to `gulp`, where the files get piped through `imagemin` before landing in `src/assets/thumbnails`:

```js
const THUMBNAIL_DIRECTORY = "posts/thumbnails";
const THUMBNAIL_OUTPUT_DIRECTORY = "src/assets/thumbnails";

export default () =>
  gulp
    .src(`${THUMBNAIL_DIRECTORY}/*`)
    .pipe(imagemin())
    .pipe(gulp.dest(THUMBNAIL_OUTPUT_DIRECTORY));
```

## Using the posts in Angular

Great! Our `files` array holds everything we need to write the HTML files, and our thumbnails are ready. Now we just have to make this available to Angular. I considered a few options:

- Serve the posts over an API endpoint.
- Dynamically generate new components for each post.

Ultimately, I decided against those approaches — deploying a whole server felt like overkill, and dynamically generating components goes against Angular best practices. Instead, I leaned on the fact that Angular builds everything into a static site. One generic `post` component plus a JSON payload would do the trick. The build script writes the `files` array to `src/posts.json` and also emits an `.html` file per post so I can visually inspect the Markdown conversion:

```js
// Write only the published: true posts to posts.json for Angular to serve
fs.writeFileSync(OUTPUT_FILE, JSON.stringify(files.filter((p) => p.data.published)));

// Emit individual HTML files for inspection
files.forEach((file) => {
  if (!file) return;

  const filename = path.join(
    OUTPUT_DIRECTORY,
    `${path.basename(file.filename, ".md")}.html`
  );

  fs.writeFileSync(filename, file.content);
});
```

From there, I generated two components: `blog` for the list view and `post` for rendering individual articles.

### But first, routing

In `app-routing.module.ts`, the relevant routes are tiny:

```ts
const routes: Routes = [
  { path: 'blog', component: BlogComponent },
  { path: 'blog/:postlink', component: PostComponent },
  { path: 'not-found', component: PageNotFoundComponent },
  { path: '**', redirectTo: '/not-found' },
];
```

### The Post component

`post.component.html` is the generic template that loads each article. The classes come from the [Bulma CSS framework](https://bulma.io):

```html
<div class="main section container pt-0">
  <div class="block is-fullwidth">
    <h1 class="title is-1">{{ meta.title }}</h1>
    <p class="has-text-grey">Published {{ meta.date }}</p>
  </div>
  <div class="content" [innerHTML]="content"></div>
</div>
```

The inline directive `[innerHTML]="content"` loads the raw HTML into the component so the post renders exactly as the Markdown produced it.

And here’s the relevant TypeScript:

```ts
import hljs from 'highlight.js';
import posts from 'src/posts.json';

interface PostMeta {
  title: string;
  date: string;
  content: string;
  thumbnail: string;
  permalink: string;
}

const postData = posts.filter(Boolean).map((p) => {
  const post: PostMeta = {
    title: p.data.title,
    date: new Date(p.data.date).toLocaleDateString('en-US', {
      month: 'long',
      day: 'numeric',
      year: 'numeric',
    }),
    content: p.content,
    thumbnail: `/assets/thumbnails/${p.data.thumbnail}`,
    permalink: p.data.permalink,
  };

  return post;
});
```

When the component initializes, it finds the matching permalink from the route. Missing posts redirect to `/not-found`:

```ts
ngOnInit(): void {
  this.postlink = this.route.snapshot.params['postlink'];
  const postMeta = postData.find((p) => p.permalink === this.postlink);

  if (postMeta) {
    this.meta = postMeta;
    this.content = postMeta.content;
  } else {
    this.router.navigate(['/not-found']);
  }
}

ngAfterViewInit(): void {
  this.document.querySelectorAll('code').forEach((el) => {
    hljs.highlightElement(el as HTMLElement);
  });
}
```

### The Blog component

The `Blog` component lists every published post using the same data source:

```html
<div class="main section container pt-0">
  <div *ngFor="let post of posts" class="box is-rounded p-0">
    <a class="nostyle" [href]="'/blog/' + post.permalink">
      <div class="columns">
        <div
          class="column is-one-fifth is-thumbnail is-hidden-mobile"
          [style]="post.thumbnail"
        ></div>
        <div class="column is-four-fifths">
          <div class="content px-3 py-1">
            <h3 class="title is-3 mb-2 no-border">{{ post.title }}</h3>
            <p class="mb-2" [innerHTML]="post.content.substring(0, 180) + '...' "></p>
            <span class="tag is-info is-light">{{ post.date }}</span>
          </div>
        </div>
      </div>
    </a>
  </div>
</div>
```

Its backing code simply maps the JSON into a friendlier structure and exposes it as `posts`:

```ts
const postData = posts.map((p) => {
  const post: PostMeta = {
    title: p.data.title,
    date: new Date(p.data.date).toLocaleDateString('en-US', {
      month: 'long',
      day: 'numeric',
      year: 'numeric',
    }),
    content: p.content,
    thumbnail:
      `background-image: url("/assets/thumbnails/${p.data.thumbnail}"); background-position: center; background-size: cover;`,
    permalink: p.data.permalink,
  };

  return post;
});

export class BlogComponent {
  posts = postData;
}
```

## Deployment

Deployment happens through AWS. Pushing to GitHub triggers CodeBuild, which installs dependencies, builds the Angular project, and syncs the static files to an S3 bucket. CloudFront sits in front so assets are cached globally. Here’s the `buildspec.yml`:

```yaml
version: 0.2

env:
  variables:
    BUCKET_NAME: peterrauscher.com
    DISTRIBUTION_ID: EDFDVBD6EXAMPLE

phases:
  pre_build:
    commands:
      - echo Installing source NPM dependencies...
      - npm install
      - npm install -g @angular/cli
      - npm install -g gulp-cli
  build:
    commands:
      - echo Build started on `date`
      - echo Compiling the dist folder...
      - npm run build
  post_build:
    commands:
      - echo Build completed on `date`
      - echo Deleting existing site files...
      - aws s3 sync dist s3://${BUCKET_NAME}/ --delete
      - echo Invalidating CloudFront cache...
      - aws cloudfront create-invalidation --distribution-id ${DISTRIBUTION_ID} --paths "/*"

artifacts:
  files:
    - "**/*"
  discard-paths: no
  base-directory: "dist/peterrauscher.com"
```
